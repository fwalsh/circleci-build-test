About This Demo

This page is supplementary content included to demonstrate CircleCI's dynamic config / path-filtering capability. It is not the "trivial web application with a single page and a button" required by the brief — that is project-a, which fetches live CircleCI status data and reveals it on button click.

The path-filtering pipeline routes changes in project-a/, project-b/, and shared/ to separate test workflows, demonstrating the same pattern that answers Part 3 of this take-home challenge.
